# DTR-NQ-WP-20260721-01 — Baseline integrity closure

## Status

`CLAIMED`

## Owner

ChatGPT autonomous development agent

## Goal

Close the NQ data-integrity and reproducibility gate before any further parameter optimization. The frozen reversal candidate must be protected from setup formation or trade simulation across unsafe market-data gaps.

## Scope

1. Preserve the existing pre-safety reference baseline and configuration.
2. Annotate one-minute and derived five-minute data with deterministic gap-state metadata.
3. Reset and invalidate setup formation when a classified gap occurs inside a session range or post-range signal sequence.
4. Reject open trades that would bridge an unsafe data gap from primary performance results.
5. Add explicit funnel counters for data-integrity rejections.
6. Add unit and regression tests for range gaps, signal-path gaps, and open-trade gaps.
7. Run CI on Python 3.11 and 3.12.
8. Perform an independent adversarial review before promotion.
9. Update changelog, roadmap, status, claim, and handover artifacts.

## Non-goals

- No parameter retuning.
- No continuation-engine implementation.
- No Dukascopy or additional-feed work.
- No change to the frozen candidate manifest unless required to expose integrity policy explicitly.
- No production-performance claim.

## Acceptance criteria

- Existing clean-data behavior remains deterministic.
- A session range containing an unsafe gap cannot produce a signal.
- A sweep/reclaim/BOS/acceptance path cannot continue across any reset boundary.
- An open trade encountering an unsafe gap is excluded and counted explicitly.
- Scheduled maintenance and standard weekend closures remain classified separately from unsafe gaps.
- Tests cover gaps within a five-minute bucket and between five-minute buckets.
- CI passes on Python 3.11 and 3.12.
- Independent review records a promotion decision and unresolved limitations.

## Risk controls

- Preserve function defaults and avoid hidden parameter changes.
- Do not silently synthesize missing bars.
- Keep gap metadata deterministic and derived only from observed timestamps.
- Report rather than conceal any result changes caused by the safety policy.
