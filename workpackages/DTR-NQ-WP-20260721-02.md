# DTR-NQ-WP-20260721-02 — Independent continuation engine

## Status

`CLAIMED`

## Owner

ChatGPT autonomous development agent

## Goal

Determine whether accepted session-range breakouts contain an independently tradable continuation edge on NQ after costs, without borrowing reversal signals, retuning the frozen reversal candidate, or combining branches prematurely.

## Strategic question

When price exits a DTR session range and demonstrates acceptance outside it, is continuation more likely than mean reversion, and under which session, direction, weekday, and market-state conditions is that decision useful?

## Scope

1. Define accepted upside and downside range breakouts from first principles.
2. Distinguish continuation from a sweep-and-reclaim reversal and from an unresolved range excursion.
3. Implement one-bar and two-bar acceptance variants.
4. Implement immediate breakout-close and first-pullback entries separately.
5. Add explicit failed-breakout invalidation.
6. Measure displacement, distance from range, VWAP alignment, efficiency ratio, and volume/ATR expansion as independent diagnostics before using them as gates.
7. Implement continuation-specific stops, targets, maximum hold, and scheduled close without changing reversal logic.
8. Instrument the full opportunity funnel and every rejection reason.
9. Evaluate development, validation, and later-research periods chronologically under `gap_policy: reject_unsafe`.
10. Report performance and opportunity coverage by session, weekday, direction, entry route, acceptance variant, and regime.
11. Run cost, slippage, parameter-neighbourhood, and walk-forward checks before candidate selection.
12. Perform an independent adversarial review before any combination with reversal.

## Initial architecture

### Event states

`IN_RANGE → BREAK_ATTEMPT → ACCEPTED_BREAKOUT → ENTRY_ELIGIBLE → OPEN_TRADE`

Failure transitions must be explicit:

- return inside the range before acceptance;
- opposite-side range break;
- excessive extension before entry;
- failed pullback hold;
- reset or unsafe data interval;
- session-specific decision window expiry.

### Baseline definitions

- Breakout: five-minute close beyond the range boundary by a configurable tick/ATR buffer.
- Acceptance: one or two consecutive closes outside the boundary, with no close back inside.
- Immediate entry: acceptance close.
- Pullback entry: first revisit of a narrow boundary band that closes or trades back outward without invalidation.
- Failed breakout: close back inside the range before entry or after entry according to an explicit stop/invalidation contract.

### Research order

1. Unfiltered structural baseline.
2. Acceptance duration.
3. Immediate versus pullback entry.
4. Failure/invalidation logic.
5. Stop and exit architecture.
6. Displacement and extension diagnostics.
7. VWAP, efficiency-ratio, volume, and ATR ablations.
8. Session and calendar attribution.
9. Walk-forward and neighbourhood stability.

## Acceptance criteria

- Continuation code is structurally separate from reversal code.
- Every run defaults to the locked gap-safe data contract.
- No setup or trade crosses a reset or unsafe interval.
- Baseline event fixtures cover upside/downside acceptance, false breaks, pullbacks, and gap resets.
- Funnel accounting reconciles opportunities, signals, rejections, and trades.
- CI passes on Python 3.11 and 3.12.
- An unfiltered baseline report exists before parameter search.
- No candidate is promoted on aggregate net R alone.
- Any promoted continuation candidate has independently positive validation and walk-forward evidence with acceptable opportunity coverage.
- Independent review records `PROMOTE`, `HOLD`, or `REJECT` before combination research.

## Risk controls

- Preserve the locked reversal manifests and expected baselines.
- Avoid combinatorial searches until each module has an ablation result.
- Prefer parameter plateaus over isolated optima.
- Record all assumptions in versioned manifests.
- Treat the current timestamp, rollover, and VWAP limitations as explicit research constraints.
